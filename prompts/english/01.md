# Prompt for Laptop Selection Generator — English

Create an HTML file in English — a fully static single-page web application with a step-by-step wizard interface (4 screens) for generating a text prompt to select a laptop without known issues.

---

## HEAD AND SEO

```html
<title>Laptop Reliability & Build Quality — AI Prompt Generator</title>
<meta name="description" content="Find laptops without overheating, throttling, poor build quality, and battery issues. Generate an AI research prompt based on real Reddit reviews and user reports.">
<meta charset="UTF-8">
<meta name="viewport" content="width=device-width, initial-scale=1.0">
<meta name="robots" content="index, follow">
<html lang="en">
<link rel="canonical" href="">
```

No extended SEO or structured data.

## TECHNICAL REQUIREMENTS

The file must be a single self-contained HTML file with embedded CSS and JavaScript. No CSS frameworks, JavaScript frameworks, libraries, or CDNs are allowed. The page must be fully static and responsive. AI assistant SVG icons must render correctly in both themes.

## QUERY PARAMS AND STATE

The state of all selected values is stored in URL query params. The `laptop` parameter may contain one or more values separated by commas — for example `laptop=MacBook+Air+M3,ThinkPad+X1+Carbon`. The `country` and `lang` parameters contain a single value or are absent. All parameters are optional. On page reload, state is restored from query params and the appropriate screen is shown. If query params are absent — screen 1 is shown.

## DESIGN SYSTEM

All colors, spacing, and typography are defined as CSS custom properties in the `:root` block. Hardcoded color values anywhere in CSS are not allowed, except in the `:root` and `[data-theme="dark"]` blocks. Theme switching is done via the `data-theme="dark"` attribute on the `<html>` element. Theme state is stored in `localStorage`. The theme toggle is located in the top-right corner on every screen.

```css
:root {
  --bg: #ffffff;
  --surface: #f5f5f5;
  --border: #e0e0e0;
  --text-primary: #111111;
  --text-secondary: #666666;
  --accent: #2563eb;
  --accent-hover: #1d4ed8;
  --accent-text: #ffffff;

}

[data-theme="dark"] {
  --bg: #0f0f0f;
  --surface: #1a1a1a;
  --border: #2a2a2a;
  --text-primary: #f0f0f0;
  --text-secondary: #999999;
  --accent: #3b82f6;

```

Typography: `font-family` — `var(--font)`, base font size — 16px, `line-height` — 1.6. `h1` — 1.75rem, font-weight 600. Subtitle — 1rem, font-weight 400, color `var(--text-secondary)`. Buttons — 0.875rem, font-weight 500. Textarea — 0.9rem, line-height 1.7, font-family monospace.

Chip buttons use `var(--surface)` as background, `var(--border)` as border, `var(--text-primary)` as text color. An active chip uses `var(--accent)` as background and `var(--accent-text)` as text color. All interactive elements use `var(--radius)` for border-radius. Transitions on color and background-color — 0.2s ease. No box-shadow except a thin 1px border on surfaces via `var(--border)`.

A progress bar at the top of each screen, 2px height, color `var(--accent)`, filled according to the current step: screen 1 — 25%, screen 2 — 50%, screen 3 — 75%, screen 4 — 100%.

## SCREEN 1 — LAPTOP SELECTION

At the top, `h1`: "Find a Reliable Laptop Without Overheating, Throttling & Build Issues". Below it, a subtitle: "Generate a research prompt for AI based on real Reddit reviews and user reports". Step indicator: "STEP 1 OF 4 · LAPTOP". Hint below the indicator: "Select one to analyze or multiple to compare".

Chip buttons are generated from `CONFIG.laptops.chips`. Multi-select is supported — any number of chips can be selected simultaneously. Clicking an active chip deselects it.

The last button is "Other". When "Other" is clicked, an input row appears below the chips with a text field (with an HTML datalist populated from `CONFIG.laptops.autocomplete`) and an "Add" button next to it. A model is added to the tag list when the "Add" button is clicked **or** when Enter is pressed in the input field. After adding, the field is cleared and stays focused for the next entry. Each added tag is displayed below the input row with a × button to remove it. If the user tries to add a model that is already present among active chips or existing tags — the addition is silently ignored. Any number of tags can be added. Values entered via Other are merged with selected chips and all stored together in the `laptop` query param separated by commas — no separate parameter for Other exists. When restoring state from query params, values that do not match any chip in `CONFIG.laptops.chips` are displayed as tags in the Other field, while the rest appear as active chips.

The "Skip" button skips the step without a selection. The "Next" button advances to screen 2 and saves all selected values in the `laptop` query param separated by commas.

## SCREEN 2 — COUNTRY SELECTION

Step indicator: "STEP 2 OF 4 · COUNTRY". Chip buttons are generated from `CONFIG.countries.chips`. Single select — only one country can be selected. The last button is "Other". When "Other" is clicked, the button becomes active (like a chip) and a text field with an HTML datalist populated from `CONFIG.countries.autocomplete` appears below the chips. There is no Add button. If the user selects any other chip — the Other button is deactivated, the field is hidden, and its content is reset. On "Next" — if the Other button is active and the field is not empty — the value from the field is used as the selected country and stored in the `country` query param. If the Other button is active but the field is empty — the step is treated as "Skip" (no value is saved). "Skip" button. "Back" button returns to screen 1 preserving state. "Next" button advances to screen 3.

---

## SCREEN 3 — RESPONSE LANGUAGE SELECTION

Step indicator: "STEP 3 OF 4 · LANGUAGE". Chip buttons are generated from `CONFIG.languages.chips` — the `label` field is displayed. Single select. When the screen loads, the browser language is detected via `navigator.language` and matched against the `code` field in `CONFIG.languages.chips`. If matched — that button receives the label "(your browser language)" and is highlighted as recommended. If not matched — no button is pre-highlighted. The last button is "Other". When "Other" is clicked, the button becomes active (like a chip) and a text field with an HTML datalist populated from `CONFIG.languages.autocomplete` appears below the chips — the `label` field values are displayed. There is no Add button. If the user selects any other chip — the Other button is deactivated, the field is hidden, and its content is reset. On "Next" — if the Other button is active and the field is not empty — the value from the field is used as the selected language and stored in the `lang` query param. If the Other button is active but the field is empty — the step is treated as "Skip" (no value is saved). "Skip" button. "Back" button returns to screen 2. "Next" button advances to screen 4.

## SCREEN 4 — RESULT

Step indicator: "STEP 4 OF 4 · YOUR PROMPT". Summary of selected parameters: Laptop (or Laptops if more than one is selected), Country, Language. If a parameter is not selected — "Not specified" is displayed. If multiple laptops are selected — all are displayed separated by commas.

Below that, a read-only textarea with the generated prompt. The textarea automatically adjusts its height to fit the content.

The "Copy Prompt" button copies the textarea content to the clipboard and temporarily changes its text to "Copied!" for 2 seconds. The "Share" button is placed next to "Copy Prompt" and has a secondary style (background `var(--surface)`, border `var(--border)`, text color `var(--text-primary)`). When "Share" is clicked, the presence of the `navigator.share` API is checked. If the API is available (primarily mobile) — the native share dialog is invoked with the current page URL including all query params. If the API is unavailable (desktop) — the current URL is copied to the clipboard and the button text temporarily changes to "Link Copied!" for 2 seconds, then returns to "Share". The "Share" button is always present on screen 4 — regardless of which parameters are filled.

The "Start Over" button resets all query params and returns to screen 1. The "Back" button returns to screen 3.

Below the textarea — the AI assistants section, generated from `CONFIG.ai_assistants`. Below that — the links section, generated from `CONFIG.placeholder_links`. At the bottom — a GitHub link from `CONFIG.github_url`.

## PROMPT GENERATION LOGIC

The generated prompt must be plain text with no emoji, no HTML, no Markdown, and no special formatting. The prompt is split into logical paragraphs separated by a blank line for easy human reading. Every prompt variant must include a strict instruction for the AI not to fabricate facts, make unsourced assumptions, or present unverified information. The prompt must require reliance solely on real user reports, discussions, and public sources. Reddit is required but not the only source. If a language is specified — the prompt includes an explicit requirement to respond in that language. If no language is specified — the response is in English.

The prompt must always include verification of the following aspects:

**Thermal issues and throttling:** overheating of the chassis and keyboard under load and during regular use, CPU and GPU throttling after sustained load, cooling system effectiveness, fan noise under various workloads (including the activation threshold), comparison of advertised versus real-world performance in sustained tasks.

**Battery and battery life:** battery degradation after 1–2 years of use, real-world battery life versus manufacturer claims, charging behavior via USB-C or a proprietary charger, availability of an 80% charge limit option to extend battery longevity.

**Build quality and physical components:** hinge issues (wobble, creaking, wear), lid or chassis flex, gap consistency, keyboard quality (key travel, stability, individual key failures), trackpad quality (accuracy, click issues, unintended activations).

**Display:** uneven backlight brightness (backlight bleed), screen flickering (PWM — frequency and presence at low brightness), glare on matte and glossy variants, color accuracy and compliance with advertised specs (sRGB, DCI-P3), dead or stuck pixels, OLED panel degradation over time if applicable.

**Wi-Fi, Bluetooth, and ports:** Wi-Fi or Bluetooth connection instability, issues with specific adapters or drivers, USB port and hub failures, Thunderbolt or USB4 problems (incorrect power delivery, peripheral incompatibility), heat around port areas.

**Webcam:** image quality under various lighting conditions, presence of a hardware privacy shutter, driver issues or incorrect recognition in video conferencing apps.

**RAM and SSD — hardware limitations:** whether RAM and storage are soldered or user-replaceable, real-world SSD read/write speeds versus advertised figures, SSD performance degradation when near-full.

**Linux support:** compatibility of Linux distributions with specific models, known driver issues (Wi-Fi, sleep and resume, power management, keyboard backlight), availability of official vendor support or certification.

**Repairability:** iFixit score if available, difficulty of DIY battery replacement, availability of original spare parts, cost and prevalence of authorized service centers.

**Software and drivers:** freezes, crashes, or BSODs after BIOS or driver updates, issues with sleep and wake behavior, instability of manufacturer utility software.

**Regional differences:** configuration and spec differences across markets, regional warranty service limitations, service center availability in the specific country.

**Time on market:** if a model has been available for less than one year — the prompt must explicitly require warning the user about the insufficient number of independent long-term reviews.

**Alternatives:** the prompt must require suggesting specific alternative models without the described issues.

### Prompt variants depending on filled fields

**If nothing is specified (or only language):** a general worldwide prompt for selecting a reliable laptop based on real discussions.

**If only country is specified (or country + language):** the prompt accounts for regional model availability, local configurations, regional warranty support, and specific user complaints from that region.

**If one model is specified (or one model + country, or one model + language, or all three fields with one model):** the prompt focuses on analyzing that specific model, checking all aspects from the list above, taking the country into account if specified.

**If multiple models are specified:** the prompt requires a comparative analysis of all listed models. For each aspect from the list above, the AI must compare the models against each other based on real user reports and public discussions — not abstract specifications. The comparison must be structured by aspect, not by model — meaning thermal performance is covered across all models first, then battery across all models, and so on. At the end, a summary recommendation must be provided indicating which model is the better choice and why, taking the country into account if specified. If independent reviews are insufficient for some models — the AI must explicitly note this.
